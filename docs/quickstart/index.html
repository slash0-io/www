quickstart

Free tier: precise egress rules from official vendor ranges, at plan time.

1 - add the provider

terraform {
  required_providers {
    egress = {
      source = "slash0-io/egress"
    }
  }
}

provider "egress" {}

2 - pull the ranges you need

One data source per service and purpose. Browse every slug and purpose in the catalog.

# ranges your workloads connect out to (egress direction)
data "egress_ranges" "stripe_api" {
  service = "stripe"
  purpose = "api"
}

# ranges a service connects to you from (ingress direction -
# webhook and agent sources belong in ingress rules)
data "egress_ranges" "github_hooks" {
  service = "github"
  purpose = "hooks"
}

3 - feed them into a security group rule

resource "aws_security_group_rule" "stripe_egress" {
  type              = "egress"
  from_port         = 443
  to_port           = 443
  protocol          = "tcp"
  cidr_blocks       = data.egress_ranges.stripe_api.ipv4_cidrs
  security_group_id = aws_security_group.app.id
  description       = "Stripe API (feed ${data.egress_ranges.stripe_api.sync_token})"
}

terraform plan, review, apply. Your egress rule now allows exactly Stripe's published API ranges - nothing else.

what you should know

  • Every range is official. The provider reads the signed public feed, which is built exclusively from each vendor's own publication - never third-party aggregators. Each service document carries the upstream URL, retrieval time, and content hash.
  • Direction matters. egress purposes are destinations for your outbound rules; ingress purposes (webhooks, agents, synthetic probes) are sources for your inbound rules. The catalog labels every purpose.
  • Data sources refresh at plan time. Ranges drift between applies - that is the free tier's known limitation. The hosted tier closes it: the same ranges as managed prefix lists, updated continuously in your account, no terraform apply required.
  • Mind your quota. Every CIDR consumes one security-group rule (default 60 per SG). The catalog shows each purpose's rule cost up front.
